◆ getJoystickDirection()

int8_t MD_Gamepad::getJoystickDirection ( switch_t  sw)

Get direction for the joystick position.

The joystick axis position is often more important than the actual value of its position. This method will tell the application whether a joystick is pushed into the positive or negative range of the analog axis.

A small deadband is applied around the zero value to avoid jittery feedback when the joystick is centered. If the time delay between reads has not expired, 0 is returned.

See also
getJoystickValue() for the joystick position value.
Parameters
swthe switch_t value for the analog axis to be checked (SW_X or SW_Y).
Returns
0 for centered, 1 for positive X or Y axis, -1 for negative X or Y axis.

◆ getJoystickValue()

int16_t MD_Gamepad::getJoystickValue ( switch_t  sw)

Get analog value of joystick position.

This method will return the zero adjusted analog value of the requested joystick axis.

The zero values are read during the call to begin and are subtracted from the actual analog reading to created +/- range of values centred around 0.

A small deadband is applied around the zero value to avoid jittery feedback when the joystick is centered. If the time delay between reads has not expired, the last value read is returned.

See also
getJoystickDirection() for the relative axis position only.
Parameters
swthe switch_t value for the analog axis to be checked (SW_X or SW_Y).
Returns
The zero adjusted analog value.
